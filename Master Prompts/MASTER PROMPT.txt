STEP 5 — SKILL MATCH API

Create:

File:

* app/api/skill-match/route.ts

Method:

* POST

Input:
{
"skills": [
{ "skill_set_name": "Coding", "level": 4 }
]
}

Use:

* company_skills_view
* company_search_view

Matching logic:

1. Receive student skills.
2. Validate that skills array exists.
3. Get skill names from student input.
4. Query company_skills_view where skill_set_name is in input skill names.
5. Group required skills by company_id.
6. Fetch company details from company_search_view.
7. For each company:

   * totalRequiredScore = sum of required_level
   * matchedScore = sum of min(student_level, required_level)
   * if student level >= required level, add skill to matched_skills
   * if student level < required level, add to gap_skills
8. match_percentage = matchedScore / totalRequiredScore * 100
9. Sort by:

   * match_percentage descending
   * student_fit_score descending
10. Return only top 5.

Response:
{
"results": [...]
}

Important:

* Do not hardcode companies.
* Do not expose raw database errors.
* Return user-friendly API errors.
* Keep all queries read-only.
